
This directory contains different types of seeds used for fuzzing, organized by their source and construction strategy.

## Directory Structure
```
FeatureKnowledgeBaseSeed/
├── merge_all_feature_seeds/
├── merge_seed/
├── 2_feature_seeds/
├── simpleSeed/
└── xxdb/
``` 
## Description

- `merge_all_feature_seeds/` :
Contains all merged seeds.
These seeds are generated by combining all available feature seeds into a single corpus.

- `merge_seed/` :
Stores feature knowledge base unary feature seeds.
Each seed is constructed based on a single feature extracted from the feature knowledge base.

- `2_feature_seeds/` :
Contains combined feature seeds, generated by fusing operators and functions.
These seeds aim to explore interactions between multiple features.

- `simpleSeed/`:
Simple merged seeds used mainly for testing.
These seeds usually achieve fast coverage growth and are useful for quick validation.

- `xxdb/`:
Database-specific seeds.
Each subdirectory contains seeds tailored for a specific DBMS.
